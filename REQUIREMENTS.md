# FamilyShopList — Требования к проекту

> Живой документ. ОБНОВЛЯТЬ при любом изменении логики приложения.
> Версия: 1.1 · Обновлён: 2026-06-15

## Назначение
PWA для семьи: общий список покупок + вишлист.
Жёсткие ограничения: полностью бесплатно (Firebase Spark, без карты),
одна кодовая база для Android и iPhone.

## Стек
React + TypeScript · Vite · vite-plugin-pwa · Tailwind CSS · React Router ·
Firebase (Firestore + Auth) · Firebase Hosting.
(Firebase Spark — Cloud Functions недоступны, вся логика на клиенте + правила Firestore.)

## Язык интерфейса
- NFR-1. Весь UI — на английском: кнопки, заголовки, подписи, тексты.
- NFR-2. Поля ввода (названия позиций, имена списков/вишлистов, unit) принимают
  любой язык без валидации языка.
- NFR-3. Заглушки экранов из каркаса (рус.) заменить на английские при сборке.
- NFR-4. Mobile-first, адаптивная вёрстка. Целевые устройства:
  Pixel 7 (~412px, Android, без выреза) и iPhone 11 (~414px, чёлка + home-indicator).
  Учитывать safe-area-insets на iOS: контент и нижняя навигация не заезжают
  под вырез и полоску жестов.
- NFR-5. Локальная разработка/тесты — через Firebase Local Emulator Suite
  (Auth + Firestore). В dev `lib/firebase.ts` подключается к эмулятору, в prod —
  к боевому Firebase. Скрипт `dev:emulate`. (Запуск эмулятора требует Java JDK.)
- NFR-6. Качество визуала: чистый современный UI, аккуратная типографика и отступы,
  приятная палитра, понятные иконки, состояния загрузки/пустых списков.

## Ключевая концепция: Алиас (space)
Алиас — независимое рабочее пространство, владеющее своими списками и вишлистом.
- Пользователь состоит в нескольких алиасах (например «Семья», «Личное», «Дача»).
- У алиаса есть создатель (owner) и участники (members).
- Все участники видят и редактируют всё внутри алиаса.
- Данные scoped по алиасу: списки/вишлист принадлежат алиасу, не пользователю.
- Участник имеет статус: active | removed. Исключение — мягкое (status=removed),
  запись участника не удаляется, доступ теряется, авторство созданных позиций
  сохраняется. Повторное приглашение возвращает статус active.

## Функциональные требования

### Авторизация
- FR-1. Вход только через Google-аккаунт (Firebase Auth). [planned]
- FR-2. При первом входе — автосоздание профиля пользователя. [planned]

### Алиасы
- FR-3. Создание алиаса (имя). Создатель = owner.
  Кнопка «+ Создать новый» — внизу дропдауна переключателя. [planned]
- FR-4. Переключение активного алиаса через дропдаун в шапке. [planned]
- FR-5. Назначение дефолтного алиаса → открывается при запуске. [planned]
- FR-6. Приглашение. Код и ссылка — это ОДНО приглашение в двух формах. [planned]
  - Owner: кнопка «Invite member» → панель со ссылкой /join/<code>, кнопкой
    Share/Copy и кодом мелким текстом (для ручного ввода).
  - Owner может «Regenerate» — отозвать старый код/ссылку.
  - Приглашаемый по ссылке: открыл → вход через Google → «Join {alias}?» → подтвердил.
  - Приглашаемый вручную: пункт «Join a space» в дропдауне алиасов → вводит код.
  - Всё на клиенте + Firestore, без Cloud Functions.
- FR-7. Список участников алиаса со статусами (active/removed). [planned]
- FR-8. Owner может исключить участника (мягко, status=removed). Данные
  и авторство исключённого сохраняются. [planned]
- FR-11. Возврат исключённого участника через повторное приглашение
  (код/ссылка) → status снова active. [planned]

### Поведение позиций (общее для Быстрого списка и Списков)
- FR-B1. Добавление позиции — на ОТДЕЛЬНОМ экране-панели, выезжающей справа
  (почти на весь экран): имя + количество + unit + список подсказок. [v1.1]
- FR-B2. Автозаполнение названия по истории ранее добавлявшихся позиций
  (в пределах алиаса). Список подсказок на экране добавления ПОСТОЯННЫЙ:
  тап по подсказке подставляет имя и НЕ закрывает экран; клик мимо не скрывает
  список (FR-B2.3). [v1.1]
- FR-B2.1. Блокировка подсказки: «удалить навсегда» (флаг `hidden`) — больше
  никогда не предлагается; повторное добавление позиции её не воскрешает. [v1.1]
- FR-B2.2. Раздел «Manage suggestions» на экране добавления: список всех
  подсказок с edit (rename) / delete / block. [v1.1]
- FR-B3. Опциональное количество (оба поля необязательны):
  - Числовое поле с кнопками «−» / «+» для быстрого изменения.
  - Поле Unit: свободный ввод текста + ЦВЕТНЫЕ кнопки быстрого выбора
    `g · kg · l · ml`; тап подставляет значение, но можно вписать своё. [v1.1]
- FR-B4. Дедупликация по названию: добавление позиции с существующим названием
  обновляет запись, а не дублирует. Если позиция была куплена — возвращается
  в «не куплено» с новыми данными. [planned]
- FR-B5. Отметка «куплено»: позиция уходит вниз в свёрнутую секцию «Купленные». [planned]
- FR-B6. Снятие чекбокса у купленной позиции возвращает её в активный список. [planned]
- FR-B7. Сортировка активных позиций — по дате добавления, новые сверху. [planned]
- FR-B8. Купленные внутри свёрнутой секции — по времени покупки. [planned]
- FR-B9. Отображение автора позиции (аватар). [planned]
