# FamilyShopList — Требования к проекту

> Живой документ. ОБНОВЛЯТЬ при любом изменении логики приложения.
> Версия: 0.2 · Обновлён: 2026-06-14

## Назначение
PWA для семьи: общий список покупок + вишлист.
Жёсткие ограничения: полностью бесплатно (Firebase Spark, без карты),
одна кодовая база для Android и iPhone.

## Стек
React + TypeScript · Vite · vite-plugin-pwa · Tailwind CSS · React Router ·
Firebase (Firestore + Auth) · Firebase Hosting.
(Firebase Spark — Cloud Functions недоступны, вся логика на клиенте + правила Firestore.)

## Ключевая концепция: Алиас (space)
Алиас — независимое рабочее пространство, владеющее своими списками и вишлистом.
- Пользователь состоит в нескольких алиасах (например «Семья», «Личное», «Дача»).
- У алиаса есть создатель (owner) и участники (members).
- Все участники видят и редактируют всё внутри алиаса.
- Данные scoped по алиасу: списки/вишлист принадлежат алиасу, не пользователю.
- Участник имеет статус: active | removed. Исключение — мягкое (status=removed),
  запись участника не удаляется, доступ теряется, авторство созданных позиций
  сохраняется. Повторное приглашение возвращает статус active.

## UX / расположение
- Шапка: переключатель алиаса (дропдаун) + аватар/профиль.
- Нижняя навигация: Покупки · Вишлист · Ещё.
- Кнопка «+ Создать новый» — внизу дропдауна переключателя алиасов.
